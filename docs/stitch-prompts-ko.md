## 프롬프트 3-B: 벌크 알람 설정 드로어

```
[컨텍스트] 리소스 페이지 "알람 설정" 벌크 액션에서 열리는 슬라이드 드로어. 확정된 디자인 시스템 사용.

[컴포넌트]
헤더: "23개 EC2 인스턴스 알람 설정" + 닫기 X 버튼

경고 배너 (조건부, 주황 배경): "같은 타입의 리소스를 선택해야 벌크 알람 설정이 가능합니다" — 혼합 타입 시 표시, 드로어 내용 비활성화.

선택적 메트릭 업데이트 테이블:
| ☐ 선택 | 토글 | 메트릭명 | 새 임계치 | 단위 | 방향 | 현재 범위 |
|--------|------|---------|----------|------|------|----------|
| ☑ | [ON] | CPU 사용률 | [90] | % | ▲ | 23개 리소스 중 70~90 |
| ☐ | — | 메모리 | — | % | ▲ | 23개 리소스 중 75~85 |
| ☑ | [ON] | 디스크 (전체) | [85] | % | ▲ | 23개 리소스 중 80~80 |
| ☐ | — | 상태 체크 | — | | ▲ | 23개 리소스 중 0~0 |

- "선택" 체크박스: 체크된 메트릭만 변경. 미체크 = 리소스별 기존 값 유지.
- 헤더 행에 "전체 선택" 체크박스
- 미체크 행: 회색 처리, 입력 비활성화
- "현재 범위": 선택된 리소스들의 min~max 임계치 (회색 텍스트)
- 방향: ▲ = 높을수록 위험, ▼ = 낮을수록 위험

오버라이드 경고 (조건부): "⚠ 23개 중 5개 리소스에 CPU 커스텀 임계치가 있으며 덮어씌워집니다"

고정 푸터:
- "변경 미리보기" 보조 버튼 → 선택된 메트릭만 리소스별 이전→이후 diff 표시
- "선택 메트릭 적용" 주요 버튼 → 확인 모달: "23개 EC2 인스턴스의 CPU, 디스크 임계치를 업데이트하시겠습니까?"

[상태]
- 혼합 타입: 경고 배너, 콘텐츠 비활성화
- 메트릭 미선택: 적용 버튼 비활성화
- 미리보기 펼침: 테이블 아래 스크롤 가능 diff 목록

```

```
[컴포넌트]
섹션: 표준 메트릭 (리소스 타입별 자동 채움)
테이블 컬럼:
| 토글 | 메트릭 | CW 메트릭(회색 모노) | 방향 | 임계치(입력) | 단위 | 등급(읽기 전용 뱃지) | 상태(필) | 현재 값 | 소스(뱃지) | 추세(스파크라인) |

EC2 예시 행:
| [ON]  | CPU          | CPUUtilization    | ▲ | [80]  | % | SEV-3      | 🟢 OK    | 23.4% | System   | ↗ 스파크라인 |
| [ON]  | 메모리       | mem_used_percent  | ▲ | [80]  | % | SEV-3      | 🟢 OK    | 45.2% | Customer | → 스파크라인 |
| [ON]  | 디스크 (/)   | disk_used_percent | ▲ | [80]  | % | SEV-3      | 🔴 ALARM | 82.1% | Custom   | ↗ 스파크라인 |
| [OFF] | 디스크 (/data)| disk_used_percent | ▲ | [80]  | % | SEV-3      | ⚫ OFF   | 55.3% | System   | → 스파크라인 |
| [ON]  | 상태 체크    | StatusCheckFailed | ▲ | [0]   |   | SEV-1      | 🟢 OK    | 0     | System   | → 스파크라인 |

- 토글: 메트릭별 ON/OFF (OFF = 해당 알람 비활성화)

- 상태: 색상 필 (OK 초록, ALARM 빨강, INSUFFICIENT 주황, OFF 회색)
- 현재 값: 모노스페이스, 임계치 초과 시 빨강
- 소스 뱃지: "System"(slate), "Customer"(파랑), "Custom"(보라) — 임계치 출처
- 추세: 작은 스파크라인(80x24px) 최근 1시간

섹션: 커스텀/동적 메트릭

  → 인라인 폼 행: 메트릭명(텍스트), 임계치(숫자), 방향(▲/▼ 드롭다운), 단위(텍스트), 저장/취소
  → 안내: "CloudWatch에 메트릭이 존재해야 합니다. list_metrics API로 네임스페이스 자동 해석."
- 기존 커스텀 알람 테이블:
  | 메트릭 | 네임스페이스 | 임계치 | 방향 | 상태 | 현재 값 | 소스 | 액션 |
  | CommitLatency | AWS/RDS | 0.05 | ▲ | 🟢 OK | 0.012s | Tag | (보기만) |
  | custom_errors | CustomApp | 10 | ▲ | 🟡 INSUF | — | UI | 편집/삭제 |
- 소스 "Tag" = 읽기 전용, "UI"/"Import" = 편집 가능

고정 액션 바 (하단, 흰 배경, border-t, p-4):
- 왼쪽: 미저장 변경 표시 (주황 점 + "미저장 변경사항") — 변경 없으면 숨김
- 오른쪽: "기본값으로 초기화"(테두리), "고객사 기본값 적용"(테두리), "변경사항 저장"(파랑 주요, 변경 없으면 비활성화)

[상태]

- 더티 (변경 있음): 주황 점, 저장 활성화
- 저장 중: 저장 버튼에 스피너

```
