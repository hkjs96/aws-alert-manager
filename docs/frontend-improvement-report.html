Major Resource Table: 정보 밀도 과부하 — 한눈에 파악 어려움 Resources
🔴 현재
  • 이름/타입/계정/리전/알람수/모니터링 토글 — 6개 컬럼이 좁은 화면에서 압축
  • Monitoring 토글 위치가 맨 마지막 — 주요 액션 접근성 낮음

  • Monitoring 토글을 Name 옆으로 이동 — 최우선 액션
  • SEV 최고 등급만 배지로 표시해 노이즈 감소

Alarms 페이지 개선
Major State 탭 필터: count 배지 없고 구조적으로 파악 어려움 Alarms
🔴 현재
  • ALL / ALARM / INSUFFICIENT / OK / OFF 탭 — 각 탭에 건수 없음
  • 탭 클릭 전에 각 상태에 몇 건인지 전혀 모름
  • 시간이 ISO 형식(2024-01-15T09:30:00Z) 그대로 — 가독성 낮음
  • 알람 Value가 단순 숫자 — 임계치 대비 얼마나 초과했는지 모름
🟢 개선안
  • 탭마다 건수 badge 추가: ALARM (4) 빨간 배지, OK (8) 녹색 배지
  • 시간: "3분 전", "2시간 전" 등 relative time 표시
  • Value 컬럼: 임계치 대비 % 표시 (예: 87% / 80% threshold)
  • ALARM 상태 행은 배경 옅은 빨강으로 강조

Resource Detail 개선
Critical Health Map: 플레이스홀더 상태 — 실제 데이터 없이 애니메이션만 Resource Detail
🔴 현재
  • 6개 의존성 바가 랜덤 애니메이션만 — 실제 상태 반영 안 됨
  • 어떤 의존성인지 라벨이 "Database", "API" 등 의미 없는 generic 텍스트
  • 클릭해도 아무 동작 없음
🟢 개선안 (Phase 1 현실적 접근)
  • 해당 리소스의 실제 메트릭별 최신 상태로 채우기 (alarm_registry 연동)
  • 각 셀 = 메트릭명 + 현재 상태(OK/ALARM/INSUFFICIENT)
  • 셀 클릭 시 해당 알람 설정으로 스크롤
  • 데이터 없으면 Health Map 섹션 전체 숨김 (빈 플레이스홀더 제거)

Settings 페이지 개선
Minor Threshold 테이블: 컬럼이 너무 많고 Customer Override 입력 UX 불편 Settings
🔴 현재
  • Metric Key / Direction / Unit / System Default / Customer Override / Active Level — 6컬럼
  • Customer Override를 바꿔도 System Default 대비 어떤 효과인지 시각적으로 안 보임
  • Resource type 탭이 EC2, RDS, S3, LAMBDA, ALB ... 많아서 어느 탭인지 파악 힘듦
🟢 개선안
  • Direction + Unit 컬럼 합치기 (≥ 80%) — 컬럼 수 감소
  • Override 입력 옆에 미니 progress bar로 시스템 기본 대비 위치 시각화
  • 변경사항이 있는 탭에 • 표시
  • 저장 버튼을 테이블 하단 고정 → 스크롤해도 보임

구현 로드맵

Phase 1 — 즉시 시작 가능 1–2일

TopBar 알람 count badge · Stat Cards 트렌드 표시 · Alarm 탭 count · 시간 relative 표기 · 각 페이지 Empty State UI

Phase 2 — 구조 개선 3–5일

Sidebar Collapsible (160px ↔ 64px) · Resource Table Status 컬럼 + 알람 배지 분리 · Health Map 실제 메트릭 Grid로 교체

Phase 3 — 완성도 1주+

모바일 반응형 (오버레이 드로어) · Settings Threshold UX 개선 · 접근성(ARIA) · 애니메이션 폴리싱
